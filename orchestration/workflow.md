# UI Foundations Workflow

## Purpose
This workflow defines how new design-system components move from request to draft, to Figma, back into the repository, and eventually toward Code Connect readiness.

The goal is not maximum automation from day one, but a clear, repeatable, understandable process that can later be automated in the right places.

## Core Principle
**Conversation is input. Draft is truth. Project is control. Figma and repository are execution layers.**
